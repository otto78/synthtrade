# Plan: Fix start.ps1 — Set-ExecutionPolicy e problemi di formattazione

## Problemi riscontrati
1. **Set-ExecutionPolicy fallisce** — Il modulo `Microsoft.PowerShell.Security` non viene caricato nella nuova istanza di PowerShell avviata da `Start-Process`, causando un errore che blocca lo script.
2. **Errori di formattazione da PowerShell esterno** — Quando lo script viene eseguito da PowerShell (non dal terminale integrato VS Code), la codifica della console non è UTF-8 e comandi come `npx ng serve` producono output corrotto.

## Causa radice
- `Set-ExecutionPolicy -Scope Process` avvia un nuovo processo PowerShell figlio senza il modulo Security caricato.
- La nuova finestra PowerShell eredita la codifica di sistema (tipicamente CP1252 su Windows italiano) invece di UTF-8.

## Soluzione proposta

### 1. Rimuovere `Set-ExecutionPolicy`
Non è necessario all'interno di uno script già in esecuzione. La policy è già applicata quando l'utente esegue lo script.

### 2. Forzare UTF-8 all'inizio dello script
Aggiungere `chcp 65001 >$null` come prima riga utile, per impostare la code page UTF-8 in tutte le finestre avviate.

### 3. Usare `pwsh` invece di `powershell`
`pwsh.exe` (PowerShell Core) gestisce meglio l'encoding e non richiede il modulo Security per policy di esecuzione base.

### 4. Modifiche specifiche su `start.ps1`

**Prima (righe 12-14):**
```powershell
Start-Process powershell -ArgumentList "-NoExit", "-Command", `
  "Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned; & '$venv'; cd '$backend'; uvicorn app.main:app --reload --port 8888" `
  -WindowStyle Normal
```

**Dopo:**
```powershell
Start-Process pwsh -ArgumentList "-NoExit", "-Command", `
  "chcp 65001 | Out-Null; & '$venv'; cd '$backend'; uvicorn app.main:app --reload --port 8888" `
  -WindowStyle Normal
```

Stessa modifica per il frontend (riga 17-19): sostituire `powershell` con `pwsh` e aggiungere `chcp 65001 | Out-Null` nel comando.

### 5. Aggiungere `chcp 65001` anche nella finestra padre
All'inizio dello script, dopo `$root = ...`:
```powershell
chcp 65001 | Out-Null
```

## Passi di implementazione
1. Editing di `start.ps1`:
   - Aggiungere `chcp 65001 | Out-Null` dopo la definizione delle variabili
   - Sostituire `powershell` → `pwsh` in entrambi gli `Start-Process`
   - Rimuovere `Set-ExecutionPolicy -Scope Process -ExecutionPolicy RemoteSigned;` dai command strings
   - Aggiungere `chcp 65001 | Out-Null;` all'inizio di ogni command string passata a pwsh
2. Verificare che `pwsh.exe` sia nel PATH (di solito lo è con PowerShell 7+ installato)
3. Testare lo script da:
   - Terminale integrato VS Code
   - PowerShell esterno (non VS Code)
